================================================================================
HOW TO CREATE ZIP FILE FOR GEMINI AI
================================================================================

You have 3 options to create the ZIP file. Choose one:

================================================================================
OPTION 1: AUTOMATIC (RECOMMENDED)
================================================================================

If you're on macOS or Linux:
  $ chmod +x create-gemini-zip.sh
  $ ./create-gemini-zip.sh

If you're on Windows:
  > create-gemini-zip.bat

The script will automatically:
- Gather all files
- Create organized folder structure
- Generate manifest
- Create ZIP file: modul-ajar-platform-gemini.zip

That's it! Skip to "UPLOAD TO GEMINI" section below.

================================================================================
OPTION 2: MANUAL USING FILE EXPLORER (EASIEST FOR BEGINNERS)
================================================================================

Windows:
  1. Right-click on empty space
  2. Click "New" → "Folder"
  3. Name it: "gemini-upload"
  4. Inside, create these folders:
     - docs/
     - src/
     - config/
  5. Copy files:
     - Copy "docs/" folder → gemini-upload/docs/
     - Copy "app/" → gemini-upload/src/app/
     - Copy "components/" → gemini-upload/src/components/
     - Copy "lib/" → gemini-upload/src/lib/
     - Copy "hooks/" → gemini-upload/src/hooks/
     - Copy "package.json", "tsconfig.json", "next.config.mjs" → gemini-upload/
  6. Select "gemini-upload" folder
  7. Right-click → "Send to" → "Compressed (zipped) folder"
  8. Done! You'll get "gemini-upload.zip"

macOS:
  1. Create folder: gemini-upload
  2. Copy files as above
  3. Right-click folder → "Compress 'gemini-upload'"
  4. Done! You'll get "gemini-upload.zip"

Linux (File Manager):
  1. Create folder: gemini-upload
  2. Copy files as above
  3. Right-click → "Compress" or "Create Archive"
  4. Choose ZIP format
  5. Done!

================================================================================
OPTION 3: COMMAND LINE
================================================================================

macOS/Linux:
  $ mkdir gemini-upload
  $ cp -r docs/ gemini-upload/
  $ mkdir gemini-upload/src
  $ cp -r app components lib hooks prisma scripts gemini-upload/src/
  $ cp package.json tsconfig.json next.config.mjs middleware.ts gemini-upload/
  $ zip -r modul-ajar-platform-gemini.zip gemini-upload/

Windows (PowerShell):
  > mkdir gemini-upload
  > xcopy docs gemini-upload\docs /E /I
  > mkdir gemini-upload\src
  > xcopy app gemini-upload\src\app /E /I
  > xcopy components gemini-upload\src\components /E /I
  > xcopy lib gemini-upload\src\lib /E /I
  > xcopy hooks gemini-upload\src\hooks /E /I
  > xcopy prisma gemini-upload\src\prisma /E /I
  > xcopy scripts gemini-upload\src\scripts /E /I
  > copy package.json gemini-upload\
  > copy tsconfig.json gemini-upload\
  > copy next.config.mjs gemini-upload\
  > copy middleware.ts gemini-upload\
  > Compress-Archive -Path gemini-upload -DestinationPath modul-ajar-platform-gemini.zip

================================================================================
TROUBLESHOOTING
================================================================================

Q: ZIP file is too large?
A: That's normal (probably 2-5 MB). Gemini can handle it.

Q: Can't create ZIP file?
A: Use Option 2 (File Explorer) or online tools like:
   - zipme.online
   - ezip.me
   - online-convert.com

Q: Gemini can't read the ZIP?
A: Extract it locally first, then copy-paste individual files or
   use the GEMINI_STARTER_PROMPT.md method instead.

Q: Which files are most important?
A: Critical for Gemini:
   1. docs/GEMINI_STARTER_PROMPT.md (read this first)
   2. docs/GEMINI_AI_CONTINUATION_PROMPT.md (dev guidelines)
   3. lib/firebase/db.ts (database layer)
   4. app/dashboard/modules/create/page.tsx (form builder)

Q: Can I modify the ZIP before uploading?
A: Yes! Feel free to:
   - Add your own notes
   - Remove unnecessary files
   - Rename folders
   - Add .env.example with your own values
